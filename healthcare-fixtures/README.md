# Bundled healthcare fixtures

These files seed the same healthcare graph the interactive demo expects.
Bundling them in the repo guarantees a fresh clone reproduces the demo
exactly — patient names, provider names, diagnoses, medications, and
their relationships are pinned, not regenerated.

## Layout

```
healthcare-fixtures/
├── data/
│   ├── fixtures.json     POLE+O entities + relationships + sample documents
│   ├── ontology.yaml     Entity types, fields, relationship vocabulary
│   └── _base.yaml        Base POLE+O classes
└── cypher/
    ├── schema.cypher          Indexes for Person/Organization/Location/Event/Object
    └── gds_projections.cypher Optional GDS native projections
```

## Reproducing the demo

The demo entry point (`scripts/seed_test_data.py`) already points here by
default. From the repo root:

```bash
# 1. Apply the Semvec graph schema + load healthcare fixtures
python scripts/seed_test_data.py

# 2. Run the interactive demo
python scripts/interactive_demo.py
```

You will see the same five patients (James Morrison, Aisha Patel,
Maria Rodriguez, Carlos Gutierrez, David Park) and five providers
(Dr. Sarah Chen, Dr. Michael O'Brien, Dr. Elena Volkov, Dr. Yuki Tanaka,
Dr. Rachel Okonkwo) the demo scenarios reference by name.

## Override

Set `FIXTURES_PATH` and `HEALTHCARE_SCHEMA_PATH` in your `.env` to point
at a different scaffold (for example one freshly generated with
`uvx create-context-graph healthcare-test-data --domain healthcare
--framework pydanticai --demo-data`).

## Provenance

Originally generated by [create-context-graph](https://github.com/neo4j-labs/create-context-graph)
in healthcare/PydanticAI mode. Only the data + schema files are tracked
here — the upstream project also produces a FastAPI backend and Next.js
frontend that the Semvec demo does not use.
